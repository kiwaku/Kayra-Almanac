## Hermes Components

| Component | Location |
|-----------|----------|
| Binary | `~/.local/bin/hermes` |
| Config | `~/.hermes/config.yaml` |
| Environment | `~/.hermes/.env` |
| Soul/Persona | `~/.hermes/SOUL.md` |
| State DB | `~/.hermes/state.db` |
| Skills | `~/.hermes/skills/` (27+ directories) |
| Memory | `~/.hermes/memories/MEMORY.md` |
| Model | `qwen3.5-35b-a3b-ud-q4_k_xl` via `http://127.0.0.1:8081/v1` |

```bash
hermes cron list                  # Show all scheduled jobs
hermes cron create --name "..." --schedule "..." --prompt "..."
hermes cron run <job_name>        # Execute immediately
```

**Watch out:** `auto_cron` declarations in `config.yaml` are declarations-only. Only entries in `jobs.json` actually run. Always verify with `hermes cron list`.

## Discord Commands (12)

| Command | Action |
|---------|--------|
| `!status` | Mode, queue depth, last health check, VRAM, uptime |
| `!search <query>` | Quick SearXNG search (ephemeral) |
| `!research <query>` | Research mode search (stores findings) |
| `!run <lane>` | Execute hunt lane immediately |
| `/hunt now` | Run all enabled hunt lanes |
| `/digest now` | Generate and deliver digest |
| `/approve <id>` | Approve proposed memory entry |
| `/reject <id>` | Reject proposed memory entry |
| `/quiet` | Zero notification budget (silence proactive messages) |
| `/observe` | Run observe.sh, return summary |
| `/lanes` | Lane config + DB stats |
| `/backup` | Run backup.sh |

In Mode A, commands needing CUDA auto-enqueue instead of executing.

**Direct message delivery** (bypasses LLM, no agent loop):
```bash
openclaw message send \
  --channel discord \
  --target "channel:1480687290928988172" \
  --message "text content here"
```

## Mode A/B

| | Mode A (CUDA offline) | Mode B (CUDA online) |
|-|----------------------|---------------------|
| Trigger | health-check fails | health-check succeeds |
| LLM commands | Auto-enqueue | Execute immediately |
| Observations | Collected + classified | Collected + classified + interpreted |
| On transition A->B | -- | drain-queue.sh --all + interpret.sh |
