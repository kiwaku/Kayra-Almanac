## Hermes Agent

| Component | Location |
|-----------|----------|
| Binary | `~/.local/bin/hermes` |
| Config | `~/.hermes/config.yaml` |
| Environment | `~/.hermes/.env` |
| Soul/Persona | `~/.hermes/SOUL.md` |
| State DB | `~/.hermes/state.db` |
| Skills | `~/.hermes/skills/` (27+ directories) |
| Memory | `~/.hermes/memories/MEMORY.md` |
| Model | `qwen3.5-35b-a3b-ud-q4_k_xl` via `http://127.0.0.1:8081/v1` |

```bash
hermes cron list                  # Show all scheduled jobs
hermes cron create --name "..." --schedule "..." --prompt "..."
hermes cron run <job_name>        # Execute immediately
```

**Gotcha:** `auto_cron` declarations in `config.yaml` are declarations-only. Only entries in `jobs.json` actually run. Always verify with `hermes cron list`.

## Infrastructure Scripts

All at `~/.openclaw/scripts/`.

### queue-db.sh (12 commands)

```bash
queue-db.sh init                  # Create all tables
queue-db.sh enqueue <lane> <payload>
queue-db.sh dequeue               # Get next pending job
queue-db.sh complete <job_id>
queue-db.sh fail <job_id>
queue-db.sh status                # Queue stats and current mode
queue-db.sh log-event <type> <data>
queue-db.sh budget-check          # Notification budget remaining
queue-db.sh budget-inc            # Increment budget counter
queue-db.sh state-get <key>
queue-db.sh state-set <key> <value>
queue-db.sh drain-ready           # Pending jobs (Mode B only)
```

### knowledge.sh (12+ commands)

```bash
knowledge.sh stats                # Node/edge counts, graph summary
knowledge.sh query <sql>          # Raw SQL
knowledge.sh add-node <type> <title> [metadata]
knowledge.sh add-edge <src_id> <tgt_id> <type>
knowledge.sh log-event <node_id> <event_type>
knowledge.sh recent [count]       # Recently created nodes
knowledge.sh search <query>       # FTS search (sorted by identity_score)
knowledge.sh find-node <name>     # Lookup by name
```

All writes use `flock` serialization to prevent SQLITE_BUSY errors.

### Observation pipeline

```bash
observe.sh                        # Orchestrator: runs collectors, merges, classifies
  app.sh                          # Active app + window title (osascript)
  browser.sh                      # URL + page title (ActivityWatch primary, osascript fallback)
  command.sh                      # Terminal commands + intent signals
  repo.sh                         # Git status of watched repos
  files.sh                        # Recently modified files
  runtime.sh                      # CUDA status, queue depth, running jobs
  system.sh                       # Memory, CPU, disk
```

Intent signals from command.sh: `coding`, `debugging`, `deploying`, `researching`, `claude_session`. Detects loops (same command 3+ times).

### Classification rules (in observe.sh)

```
ESCALATE  if  mem > 90%  or  (CUDA down AND queue > 2)  or  loop detected
DIGEST    if  mode changed  or  git branch changed  or  burst >5 events
          or  mem > 85%  or  intent = debugging/coding  or  claude session
STORE     if  dirty files > 5  or  autoresearch running  or  cmd count > 3
IGNORE    everything else
```

### Interpretation and actions

```bash
interpret.sh                      # 30-min (Mode B), dispatches:
```

| Action | Result |
|--------|--------|
| `ignore` | Marked processed |
| `store_only` | Logged to events table |
| `digest_candidate` | Inserted to findings table |
| `proactive_message` | Discord message (budget-gated, downgrades to digest if exhausted) |
| `propose_memory` | Suggest MEMORY.md entry |
| `queue_autoresearch` | Enqueue research job |
| `auto_search` | Dispatch search-router.sh |

### Search and research

```bash
search-router.sh "<query>" --mode quick     # Ephemeral SearXNG search
search-router.sh "<query>" --mode research  # Stores findings, ingests to KG
run-hunt.sh <lane>                          # Execute specific hunt lane
hunt-ingest.sh                              # Dual JSON parser -> findings table
generate-digest.sh                          # Group by lane, star ratings, format
```

### Knowledge graph operations

```bash
extract-entities.sh               # Qwen NER -> nodes + mentions edges (hourly)
detect-patterns.sh                # Co-occurrence analysis
generate-hypotheses.sh            # Graph structure -> testable hypotheses
generate-research-questions.sh    # Gap identification
export-to-obsidian.sh             # -> ~/Obsidian/Hephaestus/
  --all                           # Full re-export
  --type entity|finding|topic     # Filter by node type
  --since YYYY-MM-DD              # Only after date
```

### Scoring and ingestion

```bash
backfill-confidence.sh            # Two-pass: identity_score + activity_score
ingest-interests.sh               # Watch ~/interests/inbox/ (PDF/TXT/MD/HTML/JSON/CSV)
ingest-openai-export.py           # Import ChatGPT history from conversations.zip
ingest-conversations.sh           # Discord messages -> KG events
```

### Autoresearch

```bash
autoresearch-train.sh             # Atomic cycle:
                                  #   systemctl stop llama-server
                                  #   train (~5-7 min)
                                  #   systemctl start llama-server
                                  #   ingest results to knowledge.db + queue.db
```

### Health and lifecycle

```bash
health-check.sh                   # Probe localhost:8081/health (5-min)
                                  #   Success: mode=B, triggers drain + interpret
                                  #   Failure: mode=A, logs event
drain-queue.sh [--all]            # Process pending jobs lane-by-lane
backup.sh                         # Hot SQLite .backup, config tarball, rotate to 10
```

### Dedup

```bash
dedup-entities.sh --exact-only    # Phase 1: deterministic normalization
dedup-phase2.py                   # Phase 2: token blocking + RapidFuzz + Qwen
  --dry-run                       # Preview only
  --fuzz-threshold 90             # RapidFuzz score cutoff
  --batch-size 10                 # Pairs per LLM prompt
  --limit N                       # Cap entity count
dedup-phase2-runner.sh            # Auto-restart wrapper (CUDA lock, caffeinate, 50 retries)
```

## Database Tables

### queue.db

| Table | Key columns |
|-------|-------------|
| `jobs` | id, lane, payload, status, created_at, completed_at |
| `events` | id, type, data (JSON), timestamp |

| `findings` | id, lane, title, summary, url, relevance, dedupe_hash (UNIQUE), delivered |

| Table | Key columns |
|-------|-------------|
| `nodes` | id, type, title, metadata (JSON), identity_score, activity_score, obsidian_exported |

| `edge_events` | id, edge_id, evidence_count |

## Mode A/B

| | Mode A (CUDA offline) | Mode B (CUDA online) |
|-|----------------------|---------------------|
| Trigger | health-check fails | health-check succeeds |
| LLM commands | Auto-enqueue | Execute immediately |
| Observations | Collected + classified | Collected + classified + interpreted |
| On transition A->B | -- | drain-queue.sh --all + interpret.sh |

## Quick Operations

```bash
# Check system health
ssh cuda 'nvidia-smi'
ssh cuda 'curl -s localhost:8080/health'
queue-db.sh status

# Restart inference
ssh cuda 'sudo systemctl restart llama-server'

# Manual triggers via Discord
/observe
/hunt now
/digest now

# Egress (CUDA VM or any lab VM)
sudo egress-on       # Enable internet via bastion NAT
sudo egress-off      # Back to isolated

# SSH to CUDA VM (avoid tunnel conflicts)
```
